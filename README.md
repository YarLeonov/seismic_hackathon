# 🌍 Выделение геологических разломов 3D (Seismic Fault Detection)

## О проекте
Данный репозиторий представляет собой решение-победитель для хакатона по задаче **Сегментации и выделения геологических разломов по 3D сейсмическим данным**. 
Мы предлагаем отказоустойчивую архитектуру на базе модифицированной 3D U-Net сети, способной выявлять тонкие, прерывистые разломы в условиях сильного шума и дисбаланса данных.

## 🚀 Основные фичи нашего решения (Бизнес-ценность)
1. **Кастомная Функция Потерь (Tversky + BCE Loss)**
   В сейсмических кубах разлом занимает < 2% от всего объема. Наша Loss-функция штрафует искусственный интеллект за ложные срабатывания (False Positives), оставляя чистые геологические контуры разломов (толщиной в 1 пиксель).
2. **Оптимизированная загрузка объемов (Batched RAM)**
   Анализировать 3D кубы в памяти сложно. Мы работаем с `X.npz` форматами на лету, что экономит видеопамять инфраструктуры и позволяет быстро гонять инференс.
3. **End-to-end Pipeline**
   Проект содержит скрипты для честного подсчета `F1-score` на валидационной выборке, Production-ready `Streamlit` дашборд для интерпретации предсказаний и пакетный генератор `submit.py` для тестирования скрытых кубов.

## ⚙️ Инструкции по эксплуатации

### Шаг 1: Установка окружения
Клонируйте репозиторий и установите библиотеки (рекомендуется Python >= 3.9):
```bash
python -m venv venv
.\venv\Scripts\activate
pip install -r requirements.txt
```

### Шаг 2: Обучение модели (Training)
Поместите обучающие `.npz` кубы (с ключами `seis` и `fault`) в директорию `D:\DS 2026\seis` (редактируется в скрипте).
```bash
python baseline/train.py
```
После завершения эпох появятся веса обученной нейросети `unet3d_epoch_X.pt` в корне проекта.

### Шаг 3: Веб-Интерфейс и Инференс (Dashboard UI)
Для красивой визуализации геологом результатов и ручного тестирования:
```bash
streamlit run app/app.py
```
Позволяет загрузить любой `.npz` куб и наложить предсказанные весами (по умолчанию загружает 9-ю эпоху) маски поверх оригинальной сейсмики.

### Шаг 4: Оценка точности (Validation F1-Score)
Для расчета ключевой бизнес-метрики качества на валидации:
```bash
python baseline/evaluate.py --data_dir "D:\DS 2026\seis" --weights unet3d_epoch_9.pt
```
Скрипт вычислит F1-score на последних сегментах данных.

### Шаг 5: Генерация ответов для Leaderboard (Submit)
Чтобы прогнать всю папку *скрытых* тестовых кубов и сделать предсказания:
```bash
python baseline/submit.py --test_dir "D:\путь\к\тестам" --output_dir "submission" --weights unet3d_epoch_9.pt
```
Все `.npy` файлы предсказаний будут сложены в указанную папку вывода, готовы к отправке на платформу!
